---
title: "ArticlePage"
description: "Render one article-like content document by path and collection."
---
`HArticlePage` fetches one content document from an article-like collection and renders it with Nuxt Content. Use it for simple detail routes that should follow the shared content model.

## Usage

::ui-example{:preview='false' name="article-page-basic"}
::

When `path` is not passed, the component resolves the current route path. This is the normal setup for article detail routes where the URL and content path match.

### Explicit path

Pass `path` when the route renders a content document from a different path.

```vue [pages/articles/[slug\\].vue] [pages/articles/[slug].vue]
<script setup lang="ts">
const route = useRoute()
</script>

<template>
  <HArticlePage collection="article" :path="`/articles/${route.params.slug}`" />
</template>
```

### Custom detail layouts

Build a custom detail page when the route needs a special layout, additional data, or a different document shell. Keep repeated fetching and rendering behavior in the layer when multiple projects need the same pattern.

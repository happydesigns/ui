title: "ArticleGrid"
description: "Render a paginated grid from an article-like Nuxt Content collection."
---
`HArticleGrid` renders the result grid for article-like collections. Use it when a page should show article or event cards with pagination, optional category filtering, sorting, status filtering, and resolved category or author data.

Use `HArticleGrid` when the page already owns the surrounding navigation or filter controls. Use `HArticleList` when the page should also render the category navigation that belongs to the configured collection.

## Usage

::

The component reads collection defaults from the variants config, but page props can still override the collection, pagination size, status filter, and sorting for one route.

### Custom filters

Use `where`, `status`, and `sort` when a route needs a more specific list than the collection default.

```vue [pages/articles/featured.vue]
<template>
  <HArticleGrid
    collection="article"
    :where="[{ field: 'featured', operator: '=', value: true }]"
    :sort="{ field: 'date', direction: 'DESC' }"
    :items-per-page="3"
  />
</template>
```

### Custom date output

The grid owns fetching, empty states, pagination, and card layout. Use the `date` slot only when the card needs a different date display.

```vue [pages/events/index.vue]
<template>
  <HArticleGrid collection="event">
    <template #date="{ article }">
      <HDates :date="article.date" :date-end="article.dateEnd" />
    </template>
  </HArticleGrid>
</template>
```

title: "Public components"
description: "Understand when to use the public H* component groups provided by @happydesigns/ui."
---
The public components are prefixed with `H` to separate happydesigns layer components from upstream Nuxt UI components. Use them when a project needs shared content behavior, layer conventions, or app config integration.

Start with the highest-level component that matches the job. Use `HArticleList` or `HArticlePage` before composing article traits manually, and use Nuxt UI directly when the project only needs a generic button, card, modal, or form control.

The component pages follow the same order: usage first, generated API second, source last. The prose explains the layer behavior; the generated tables show the exact props and slots from source.

## Choosing a component

| If you need to build                                | Start with                                                                                                   | Use lower-level traits when                                                                           |
| --------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------- |
| An article or event index with category navigation. | [`HArticleList`](/en/components/article/article-list)                                                           | The route needs custom layout around the same list behavior.                                          |
| A result grid without category navigation.          | [`HArticleGrid`](/en/components/article/article-grid)                                                           | The page already owns filtering or navigation.                                                        |
| A full article or event detail page.                | [`HArticlePage`](/en/components/article/article-page)                                                           | The project needs a custom detail layout but still wants shared date, author, copy, or link behavior. |
| Reusable Markdown fragments or media sequences.     | [`HSnippet`](/en/components/content/snippet), [`HCarousel`](/en/components/content/carousel)                       | The project owns the surrounding content layout.                                                      |
| Footer navigation and actions.                      | [`HFooterColumns`](/en/components/footer/footer-columns), [`HFooterButtons`](/en/components/footer/footer-buttons) | The footer has a custom layout but should keep shared link/button conventions.                        |
| A production entry point for content editors.       | [`HStudioLoginButton`](/en/components/studio/studio-login-button)                                                  | The project needs a different Studio entry point or owns its authentication UI.                       |

## What not to document here

These pages document `@happydesigns/ui` layer behavior. They do not repeat the full Nuxt UI API for underlying components such as `UButton`, `UBlogPost`, `UCarousel`, or `UFooterColumns`. When an `H*` component forwards upstream props, use the generated prop table here for the layer surface and the Nuxt UI docs for the upstream component behavior.

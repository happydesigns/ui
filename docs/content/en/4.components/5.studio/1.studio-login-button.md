---
title: "StudioLoginButton"
description: "Expose the configured Nuxt Studio login route to unauthenticated editors."
---
`HStudioLoginButton` renders a fixed login action in the lower-left corner of a production site. It reads the configured Nuxt Studio route, validates an existing Studio session, and stays hidden in development mode or while an editor is authenticated.

Add it once to the project's root app shell. The component is client-only, so consumers do not need to wrap it in `ClientOnly`.

```vue [app/app.vue]
<template>
  <UApp>
    <NuxtPage />
    <HStudioLoginButton aria-label="Open the content editor" />
  </UApp>
</template>
```

The component requires the `nuxt-studio` module, which is already included by `@happydesigns/ui`. Production projects still need their repository and authentication settings as described in the [Nuxt Studio authentication documentation](https://nuxt.studio/auth-providers).

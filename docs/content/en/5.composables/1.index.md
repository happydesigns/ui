---
title: "Shared logic"
description: "Use shared composables when a project needs custom rendering with layer-owned behavior."
---
The composables provide the reusable logic behind the public components. Use them when a project needs custom rendering but should keep shared fetching, filtering, SEO, badge, or clipboard behavior.

| Composable            | Use                                                                                  |
| --------------------- | ------------------------------------------------------------------------------------ |
| `useArticleList`      | Fetch paginated article-like collection data with filters and resolved display data. |
| `useArticleListQuery` | Keep list pagination and category filters in the URL query.                          |
| `usePageContent`      | Fetch one content document by route path or explicit path.                           |
| `usePageSeo`          | Apply SEO metadata from content data.                                                |
| `useCategoryBadge`    | Resolve category badge config from variant app config.                               |
| `useClipboard`        | Copy text and optionally show a Nuxt UI toast.                                       |

Prefer the public components when their rendering fits. Reach for composables when the project needs a different template but should keep the same data rules.

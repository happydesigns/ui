---
title: "Layer structure"
description: "How @happydesigns/ui provides Nuxt files to projects that extend it."
---
The package is a Nuxt layer exported from its root `nuxt.config.ts`. Projects extend the package and receive the layer files as part of their Nuxt app.

```ts
export default defineNuxtConfig({
  extends: ['@happydesigns/ui'],
})
```

## Source areas

| Source              | Role                                                                                      |
| ------------------- | ----------------------------------------------------------------------------------------- |
| `app/components`    | Public `H*` components and small trait components.                                        |
| `app/composables`   | Shared composables for content fetching, list state, SEO, badges, and clipboard behavior. |
| `app/assets`        | Shared CSS entry points and layer-level styling.                                          |
| `schemas`           | Nuxt Content collection and trait schemas.                                                |
| `content.config.ts` | Registers the package collections for Nuxt Content.                                       |
| `nuxt.config.ts`    | Layer module setup, component registration, variant registry, and package entry.          |

Projects can override or add Nuxt files in their own app, but that should be a project decision. If an override becomes a repeated pattern, move the shared part into the layer.

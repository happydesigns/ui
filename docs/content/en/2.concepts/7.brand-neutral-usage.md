---
title: Brand-neutral usage
description: Keep @happydesigns/ui reusable while applying project-specific identity.
---

The layer should remain brand-neutral. It can provide the component structure, configuration hooks, and semantic defaults that projects need, but it should not hard-code one brand as the only valid expression.

## Apply branding through configuration

Use project-level app config, CSS, assets, and local components to apply a brand. The reusable component should still describe the same interface behavior after the brand changes.

Good brand-specific inputs include:

- Logo assets.
- Header and footer links.
- Category labels and badge colors.
- Typography and color tokens.
- Project-specific prose and images.

## Keep behavior stable

Branding should not change what a user is allowed to do, which workflow runs, or which domain rule applies. Those decisions belong to the app or product that owns the behavior, not to a style layer.

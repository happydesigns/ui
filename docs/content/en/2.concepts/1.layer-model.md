---
title: Layer model
description: Understand how @happydesigns/ui separates reusable layer structure from project implementation.
---

`@happydesigns/ui` gives projects a shared starting point without turning every project into the same website. The layer provides reusable structure. Each project decides which routes, content, branding, and configuration it needs.

The important distinction is between foundation and implementation. The layer defines reusable patterns for pages, article-like collections, traits, footers, and configuration. A project uses those patterns to build a concrete website, docs site, or product app.

## Concept map

| Concept | Role |
| --- | --- |
| Layer structure | How Nuxt files are provided by the package and overridden by projects. |
| App shell and layouts | How shared page structure is made available without owning every route. |
| Variants model | How features and defaults are composed for content types. |
| Content collections | How Nuxt Content documents become typed pages, articles, events, snippets, and users. |
| Brand-neutral usage | How projects apply identity and style without changing product behavior. |

Read these pages before changing the source structure of the package. They explain why the layer is split the way it is.

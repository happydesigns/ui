The layer defines collections for common happydesigns content types. Projects add files to those collections and configure rendering through app config and variants.

## Generated collections

:ui-collection-table

## Collection purpose

| Collection | Use                                                                   |
| ---------- | --------------------------------------------------------------------- |
| `page`     | General page content rendered by project routes.                      |
| `article`  | Blog, news, update, or knowledge articles with list and detail views. |
| `event`    | Dated entries with location and link support.                         |
| `snippet`  | Reusable content fragments rendered inside pages or components.       |
| `user`     | Author and person data used by article traits.                        |

The content pages document the fields expected by each collection. Use the generated table here to confirm how the package currently registers them.

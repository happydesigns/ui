---
title: "App shell and layouts"
description: "How the layer supports shared app structure without owning every project page."
---
The layer provides shared building blocks for page structure, but projects still own their routes and final app shell. This keeps the package reusable for different website types and product apps.

## What the layer provides

- Shared components for article-like pages, lists, footers, snippets, and content traits.
- App config keys for header links, footer links, GitHub source links, dates, and icons.
- Defaults for collection behavior, list rendering, and trait rendering.

## What projects provide

- The route tree.
- The project homepage and custom pages.
- The final layout composition.
- Brand assets, project CSS, and project-specific content.

This split lets a project use the same content and UI foundation without being forced into one fixed app structure.

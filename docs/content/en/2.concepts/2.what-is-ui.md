---
title: "What happydesigns/ui is"
description: "The role of @happydesigns/ui in the happydesigns ecosystem."
---
`@happydesigns/ui` is the reusable interface foundation for happydesigns projects. It packages shared Nuxt setup, Nuxt UI conventions, content collections, components, composables, and configuration keys.

Use it when a project needs the same article lists, event lists, page rendering, snippets, footer structure, content traits, or app config conventions as other happydesigns projects.

## What it is not

`@happydesigns/ui` is not a brand implementation and not a complete website. It should not own project copy, customer-specific routes, final brand assets, deployment settings, or business workflows.

It also does not replace Nuxt UI documentation. When a component is inherited directly from Nuxt UI, use Nuxt UI docs for the upstream component API and these docs for happydesigns-specific wrapping, defaults, and conventions.

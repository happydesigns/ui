---
title: "Pages"
description: "Use the page collection for general route content."
---
Use `page` for general Markdown-driven pages. A project decides the route structure, while the layer provides the schema traits and rendering helpers that keep page behavior consistent.

## Use pages for

- Static content pages.
- Landing sections that still benefit from Nuxt Content.
- Docs-like content inside a project.
- Route content that needs title, description, layout, table-of-contents, or header behavior.

## Keep out of page frontmatter

Do not encode product state, permissions, workflow decisions, or deployment settings in page frontmatter. Page frontmatter should describe content and presentation, not own domain behavior.

## Related APIs

`HArticlePage` is for article-like detail pages. For generic pages, projects can use Nuxt Content rendering directly and still use the shared schemas and `usePageSeo` where useful.

---
title: "Frontmatter patterns"
description: "Decide which data belongs in frontmatter, app config, or code."
---
Frontmatter should describe a single content document. App config should describe project-wide defaults. Code should own behavior.

| Put it in      | When                                                                 |
| -------------- | -------------------------------------------------------------------- |
| Frontmatter    | The value describes one page, article, event, snippet, or user.      |
| App config     | The value is reused across many documents in one project.            |
| Layer defaults | The value is a reusable default across happydesigns projects.        |
| Code           | The value changes behavior, data access, workflows, or domain rules. |

## Examples

- An article `category` belongs in article frontmatter.
- The visual badge for that category belongs in app config.
- The generic category badge lookup belongs in `@happydesigns/ui`.
- Permission rules for who can publish an article do not belong in frontmatter.

`@happydesigns/ui` standardizes the content shapes used by happydesigns websites and product docs. The goal is not to make every project publish the same content, but to make common content types predictable.

Use these pages when you create content files, wire list pages, or decide whether a field belongs in frontmatter, app config, or project code.

## Collections

| Collection | Use it for |
| --- | --- |
| `page` | General page content and custom route content. |
| `article` | Blog posts, updates, guides, and article-like entries. |
| `event` | Dated entries with location and link support. |
| `snippet` | Reusable content fragments embedded in pages or components. |
| `user` | Authors and people referenced from content. |

title: Installation

---

Install the package in the Nuxt project that will extend the layer.

```bash
pnpm add @happydesigns/ui
```

Then run Nuxt preparation after changing dependencies or layer configuration.

```bash
pnpm exec nuxt prepare
```

## Required peer setup

The layer is built for Nuxt 4, Nuxt UI 4, Nuxt Content 3, and the happydesigns variants module. A project that extends it should already be a Nuxt app and should keep its lockfile committed after installation.

The package exports the layer from `nuxt.config.ts`, so projects extend the package name rather than importing runtime files manually.

## Package entry points

| Entry | Use |
| --- | --- |
| `@happydesigns/ui` | Nuxt layer entry. |
| `@happydesigns/ui/schemas` | Content schema exports for projects that need to compose or inspect the collection model. |

Do not import internal files from `app/` or `schemas/` by path. If a project needs something that is not exported, add a stable export instead of depending on repository layout.

---
title: Project setup checklist
description: Check the minimum setup expected from a project that extends @happydesigns/ui.
---

Use this checklist after adding the layer or before updating a project to a new version.

1. `nuxt.config.ts` extends `@happydesigns/ui`.
2. `pnpm exec nuxt prepare` has been run after dependency or config changes.
3. `app.config.ts` defines project links, date formatting, GitHub metadata, and collection defaults where needed.
4. Content files use the collection fields documented in the content section.
5. Article-like pages use `HArticleList`, `HArticleGrid`, or `HArticlePage` before introducing custom list behavior.
6. Footer UI uses `HFooterColumns`, `HFooterButtons`, and app config links before custom footer layout is added.
7. Project branding is applied through app config, CSS, assets, and local layers, not by changing reusable component behavior.
8. Type checks run cleanly after content schemas or variants change.

The checklist is intentionally short. It catches setup drift without turning this documentation into a project migration log.

---
title: Upgrade notes
description: Upgrade @happydesigns/ui without losing project-specific ownership boundaries.
---

Upgrade the package like any other Nuxt layer dependency, then regenerate Nuxt types and run the project checks.

```bash
pnpm update @happydesigns/ui
pnpm exec nuxt prepare
pnpm run build
```

## What to check

Review these areas after an upgrade:

| Area | Check |
| --- | --- |
| Content collections | New required fields, changed defaults, and variant inheritance. |
| App config | Footer links, date formatting, GitHub metadata, category badges, and list defaults. |
| Components | Public prop names, slot names, and generated component metadata. |
| Composables | Async data keys, lazy behavior, query synchronization, and returned refs. |
| Styling | Nuxt UI defaults, project CSS, and brand-specific overrides. |

Project-specific workarounds are a signal to inspect the layer. If the workaround solves a repeated problem, promote the stable part into `@happydesigns/ui`; otherwise keep it local and explicit.

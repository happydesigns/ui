`@happydesigns/ui` is a Nuxt layer. A project extends it, configures the parts it needs, and keeps project-specific content and branding in the project that owns them.

Use this page when you add the layer to a new project or check whether an existing project still follows the expected setup.

## The shortest path

1. Install `@happydesigns/ui` in the Nuxt project.
2. Extend the layer from the project `nuxt.config.ts`.
3. Add the content collections the project uses.
4. Configure app links, date formatting, GitHub metadata, and variant defaults in `app.config.ts`.
5. Use the `H*` components and composables from the layer instead of recreating article, event, footer, and trait behavior locally.

## What this layer owns

| Area | Owned by `@happydesigns/ui` |
| --- | --- |
| Nuxt foundation | Shared layer setup, Nuxt UI integration, content conventions, and reusable layouts. |
| Content model | Page, article, event, snippet, and user collection shape. |
| Variants | Reusable feature flags and default config for article-like and page-like content. |
| Components | Public `H*` components for lists, pages, traits, snippets, carousels, and footers. |
| Composables | Shared content fetching, list query state, SEO, badges, and clipboard helpers. |

## What stays in the project

Project content, project routes, project branding, customer-specific links, and deployment settings stay in the project that ships the site or app. If a local pattern becomes useful across multiple projects, promote the shared part into the layer and keep the project-specific part outside it.

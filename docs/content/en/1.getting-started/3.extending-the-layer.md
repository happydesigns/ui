---
title: "Extending the layer"
description: "Configure a Nuxt project to use @happydesigns/ui as its shared foundation."
---
Add the layer to the project `nuxt.config.ts`.

```ts
export default defineNuxtConfig({
  extends: ['@happydesigns/ui'],
})
```

The project can then use public components such as `HArticleList`, `HArticleGrid`, `HArticlePage`, `HFooterColumns`, and `HSnippet` directly in Vue files and Markdown-rendered content where Nuxt component usage is enabled.

## Project ownership

The layer should provide reusable behavior. The project should provide the concrete site shape:

| Belongs in the project                        | Belongs in the layer                             |
| --------------------------------------------- | ------------------------------------------------ |
| Routes and pages for one website or app.      | Reusable layouts and components.                 |
| Brand assets and customer-specific copy.      | Brand-neutral component structure.               |
| Project-specific categories and footer links. | Shared config keys and default behavior.         |
| Deployment and environment bindings.          | Package-level Nuxt, content, and UI conventions. |

If a local route or component only makes sense for one project, keep it local. If the same solution repeats across several projects, move the reusable part into `@happydesigns/ui`.

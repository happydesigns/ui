---
title: Header and footer links
description: Configure project navigation and footer links.
---

Header and footer links are project-level content. The layer provides the expected config shape and footer components; the project decides which links it publishes.

## Header links

Use `app.links.header` for primary project navigation. Keep labels short and route targets stable.

## Footer links

Use `app.links.footer` with `HFooterColumns` for footer columns. The footer can receive columns directly as props, but app config is usually better for project-wide footer navigation.

```ts
export default defineAppConfig({
  app: {
    links: {
      footer: [
        {
          label: 'Product',
          children: [
            { label: 'Docs', to: '/docs' },
          ],
        },
      ],
    },
  },
})
```

Keep legal, contact, social, and source links in app config unless they are unique to one page.

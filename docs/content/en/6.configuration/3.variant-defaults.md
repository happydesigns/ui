---
title: "Variant defaults"
description: "Configure collection and trait defaults through the variants model."
---
Variant defaults let a project configure repeated behavior for content types without rewriting components. Article and event lists can share pagination, status filters, sorting, category badges, breadcrumbs, and trait defaults.

## Example shape

```ts
export default defineAppConfig({
  variants: {
    article: {
      config: {
        query: {
          fields: ['title', 'description', 'status', 'date'],
          order: { field: 'date', direction: 'DESC' },
          where: [{ field: 'status', operator: '=', value: 'published' }],
        },
        categories: {
          release: { label: 'Release', color: 'primary' },
        },
      },
    },
  },
})
```

Use variants for repeatable content presentation and query defaults. Do not use them for authorization or domain rules.

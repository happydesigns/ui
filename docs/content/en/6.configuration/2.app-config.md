`app.config.ts` is the main project configuration surface. Use it for values that affect rendering across the project but are not part of the reusable package API.

## Common keys

| Key | Use |
| --- | --- |
| `app.meta` | Copyright, social links, and GitHub source metadata. |
| `app.links.header` | Header navigation links. |
| `app.links.footer` | Footer columns and links. |
| `app.icons` | Project icon names used by content or components. |
| `app.toc` | Table-of-contents labels and behavior. |
| `app.date` | Date and datetime formatting defaults. |
| `variants` | Collection and trait defaults consumed by `useVariant`. |
| `ui` | Nuxt UI app config defaults. |

## Rule of thumb

If a value changes between projects but the component behavior stays the same, it usually belongs in `app.config.ts`. If changing the value changes the meaning of the feature, inspect whether the logic belongs in a product app or in the reusable layer.

---
title: Content query config
description: Configure default queries for article-like collections.
---

Article-like lists use query defaults from variant config. This keeps repeated list behavior out of every page component.

## Common fields

| Field | Use |
| --- | --- |
| `fields` | Select the fields needed for list rendering. |
| `order` | Set the default sort field and direction. |
| `where` | Set default filters such as published status. |
| `list.itemsPerPage` | Set default pagination size. |
| `list.labelAll` | Set the category label that means no category filter. |
| `list.noResultsMessage` | Set empty-state copy. |
| `list.noResultsIcon` | Set empty-state icon. |

Use props on `HArticleList` or `HArticleGrid` for one page override. Use variant config when several pages or the whole project should share the same defaults.

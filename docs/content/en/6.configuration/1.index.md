Most project-level behavior is configured through `app.config.ts`. The package provides keys and defaults; the project provides concrete links, labels, categories, date formats, and source metadata.

Use this section when a value should be changed for one project without changing the reusable layer.

## Configuration areas

| Area | Use |
| --- | --- |
| `app.config.ts` | Project defaults for app links, metadata, icons, dates, variants, and Nuxt UI defaults. |
| Variant defaults | Collection-specific list, badge, user, and trait behavior. |
| Header and footer links | Project navigation and footer structure. |
| Date formatting | Locale-aware date and datetime output. |
| GitHub links | Edit/source links for content-driven pages. |
| Content query config | Default filtering, sorting, and selected fields for article-like lists. |

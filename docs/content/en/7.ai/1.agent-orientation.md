These docs are written for humans and agents. Agents should use the documentation to understand intent, then inspect the source when implementation details matter.

The generated reference blocks are useful because they come from source files. They do not replace human explanation and they do not define product strategy by themselves.

## Working with these docs

1. Read the relevant concept or configuration page.
2. Inspect generated component, variant, or collection reference data.
3. Open the linked source file when changing implementation.
4. Keep project-specific behavior out of the reusable layer unless it is a repeated foundation.
5. Validate with Nuxt preparation, type checks, linting, and builds where relevant.

---
title: "Agent workflow"
description: "How agents should work with @happydesigns/ui."
---
Agents should treat `@happydesigns/ui` as a reusable foundation. A change is not finished because it compiles once; it should also fit the ownership model.

## Before editing

1. Read the page for the area being changed.
2. Inspect the source file linked from generated reference data.
3. Check whether the change belongs in the layer or in a project.
4. Search for existing patterns before adding a new abstraction.

## After editing

1. Run Nuxt preparation if components, schemas, variants, or docs config changed.
2. Run linting and builds that match the touched area.
3. Check that public component docs still resolve generated props, slots, and source links.
4. Report files changed, validation run, and any limitation.

Do not copy Nuxt UI docs into these docs. Link to upstream documentation when the behavior belongs to Nuxt UI itself.

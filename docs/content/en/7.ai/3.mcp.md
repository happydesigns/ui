Use the Docus MCP server when an agent can query the docs directly instead of relying on memory. MCP is useful for finding pages, reading current guidance, and checking generated reference routes.

MCP requires a server-capable Docus deployment. Static builds still publish pages, skills, `/llms.txt`, and `/llms-full.txt`, but they do not run a live MCP server.

Agents should still inspect the repository before editing code. The docs explain the intended model; the source code shows the implementation that will actually run.

## Good MCP questions

- Which component should I use for article lists?
- Where are variants documented?
- What does the docs say about app config ownership?
- Which page links to generated collection metadata?
